# 04 · UI Context — 文字描述版（无 Figma 设计稿）

> **生成日期**：2026-05-19
> **阶段**：Phase 4 · 自治模式
> **基于**：PRD 验收标准 + ADR 架构决策
> **方式**：因无 Figma，按 PRD 文字描述实现 + 推荐 Shadcn/ui 现成组件

---

## 设计风格基线

- **设计语言**：Linear-like 极简、信息密度高、键盘友好
- **配色**：浅色主题 + 深色主题切换（`prefers-color-scheme` 自动）
- **主色**：Anthropic 橙 `#CC785C`（向 Claude 致敬）+ 中性灰 + 危险红 + 成功绿
- **字体**：Inter（拉丁）+ 系统默认中文（思源黑体 / PingFang）
- **圆角**：4px（紧凑场景）/ 8px（卡片）
- **图标**：Lucide React（开源、Linear 风）

## 关键交互流程

### Flow A — 首次使用 Onboarding

```
1. 安装后自动打开 Options Page
   → 步骤 1: 设置主密码（PBKDF2 加密 API key 用）
   → 步骤 2: 输入 Anthropic API key + OpenAI API key
   → 步骤 3: 创建第一个项目（含上传文档）
   → 完成 → 提示 "去找一个报名页面试试"
```

### Flow B — 日常报名（核心流）

```
1. 打开任意报名页（金数据/Devpost/...）
2. 点插件图标 → Popup 自动检测到表单 → 提示"打开主面板"
3. Side Panel 打开 → 步骤 1：确认活动背景 → 步骤 2：生成草稿 → 步骤 3：提交后沉淀
4. 全过程 < 15 min
```

### Flow C — 回看历史

```
1. Options Page → 经验库 → 选某个项目
2. 显示该项目所有历史报名（时间倒序）
3. 点某条 → 显示 markdown 全文 + 标记/排除 RAG 操作
```

## 组件清单（用 shadcn/ui 现成）

| 组件 | 用途 |
|------|------|
| `Button`, `Input`, `Textarea` | 通用 |
| `Card` | 项目卡、字段卡 |
| `Sheet` | Side Panel 容器 |
| `Tabs` | Options Page 导航 |
| `Dialog` | 主密码输入弹窗、活动背景编辑 |
| `Select` | 项目切换、模型切换 |
| `Toast` | API 失败提示、保存成功 |
| `Progress` | 文档解析进度 / Streaming 草稿进度 |
| `Skeleton` | 草稿生成中占位 |
| `Badge` | 字段约束标签（"≤200 字" / "必填"） |
| `Tooltip` | 解释"为什么 AI 这么填" |
| `Accordion` | 字段列表折叠 |

## 中英双语 i18n 备忘

V1 默认中文 UI，但所有用户可见文本走 i18n key（如 `t('btn.fillAll')`）。
- 中文 messages：完整
- 英文 messages：先空白预留，PM 决定要不要做时再补

i18n 库：`react-i18next`（生态最稳）

## 可访问性（a11y）速查

- 所有交互元素 `aria-label`
- Side Panel 主操作支持纯键盘（Tab/Enter/Esc）
- 对比度满足 WCAG AA
- 状态变化用 `aria-live="polite"` 朗读（屏幕阅读器）
- 不依赖颜色单独传达信息（错误同时配 ⚠️ 图标）
